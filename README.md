# RxMedia Agency Pipeline

An AI-orchestrated workflow system for a digital marketing agency. Claude agents drive each stage of the client lifecycle — from onboarding through Webflow website delivery — with humans only reviewing and approving at defined gates.

**Central source of truth:** Notion (relational databases per client)
**Orchestration engine:** Claude (Anthropic API, claude-sonnet-4-6)
**Internal team agent:** Rex — Slack bot deployed on Railway

---

## Quick Start

```bash
# Set up environment
cp .env.example .env  # fill in API keys
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# Provision a new client (processes onboarding form → Notion + ClickUp)
make onboard

# Run any pipeline stage
make transcript CLIENT=client_key
make sitemap    CLIENT=client_key
make content    CLIENT=client_key
```

Run `make help` for a full command reference.

## Architecture

### Agent Design

All agents follow the same pattern:

1. `BaseAgent` (`src/agents/base_agent.py`) — abstract class with a `_run_tool_loop()` that handles the Anthropic tool-use loop (call → tool_use → execute → loop)
2. Sub-agents implement `_execute_tool()` and `run()`. They read all client context from Notion at runtime — no client-specific content is hardcoded in prompts.
3. `OrchestratorAgent` is a pure state machine with no LLM calls. It reads pipeline state, routes to the correct sub-agent, writes output to Notion, and advances the stage.

### Client Registry

All client config lives in `config/clients.py` as the `CLIENTS` dict. Two sources are merged at import time:

- `_MANUAL` — existing clients managed by hand (WellWell lives here permanently)
- `config/clients.json` — auto-generated by the onboarding agent via `make onboard`

Every script imports `from config.clients import CLIENTS`. No client-specific content is hardcoded anywhere.

### Notion API

**Important:** `notion-client` v3.0.0 broke `databases.create`, `databases.update`, and `databases.query`. All Notion operations use `notion._client.request()` directly (raw HTTP). Never use the high-level notion-client SDK methods for these three operations.

### Approval Gates

The pipeline pauses at each deliverable:

```
Sitemap Draft     → team reviews → client approves → Content
Content Draft     → team reviews → client approves → Stock Photos / Images
[Webflow Build]   → client reviews staging site   → Launch
```

`make mark-pending CLIENT=x` — sets Notion Stage Status to "Pending Review" + creates ClickUp task
`make advance CLIENT=x` — checks Notion approval status → runs next stage if approved

## Pipeline Stages

### Website Build

| Command | What it does |
|---|---|
| `make onboard` | Process form submission → provision Notion DBs + ClickUp folder + client brief |
| `make transcript` | Parse Gemini meeting transcript → brand prefs, decisions, action items |
| `make sitemap` | Generate page hierarchy + section outlines → Notion |
| `make content` | Per-page copy + SEO fields → Notion Content DB |
| `make stock-images` | Pexels photo curation → HTML report |
| `make images-brand` | ~15 brand AI images via Replicate → Notion |
| `make images-pages` | ~3 images per page via Replicate → Notion |

### SEO (sold separately)

| Command | What it does |
|---|---|
| `make seo-activate` | Activate SEO retainer — set GSC/GA4/GBP IDs |
| `make keyword-research` | Claude seeds → DataForSEO validates → Notion Keywords DB |
| `make competitor-research` | SERP analysis → Competitors DB |
| `make battle-plan` | SEO strategy → Notion |
| `make seo-baseline` | 90-day baseline report → Notion + HTML |

### Monthly Recurring

| Command | What it does |
|---|---|
| `make gbp-posts` | 3 GBP post drafts → Notion GBP Posts DB |
| `make seo-report` | Monthly performance report → Notion + HTML |
| `make care-plan` | PageSpeed + ADA report → Notion Care Plan DB |

### Blog Pipeline

| Command | What it does |
|---|---|
| `make blog-setup` | Create Blog Voice & Author Setup page in Notion (one-time) |
| `make blog-ideas` | Generate 20 ideas → Blog Posts DB (Status: Idea) |
| `make blog-write` | Write full posts for Approved ideas (Status: Draft) |
| `make blog-publish COMMIT=1` | Push Scheduled posts → Webflow Blog CMS collection |

## Rex — Internal Slack Agent

Rex answers questions and triggers pipeline stages from Slack. Deployed on Railway.

```
DM Rex:     "Rex, what's the pipeline status for Summit?"
            "Rex, run keyword research for Summit"
            "Rex, show me high-threat competitors for Summit"
            "Rex, create a ClickUp task for Andrea — due Friday"
```

**Architecture:** FastAPI + slack-bolt + Claude tool-use loop. Tool handlers are split into:
- `rex/tools/notion_tools.py` — Notion data lookups
- `rex/tools/clickup_tools.py` — ClickUp workspace + task management
- `rex/tools/pipeline_tools.py` — Background stage runner (posts back to Slack when done)

**Deploy:** Edit `rex/` → commit → push → Railway auto-deploys in ~30s.

## Key Conventions

- **Client key format:** lowercase snake_case (`summit_therapy`, `wellwell`)
- **All Notion calls:** use `notion._client.request()` — never high-level SDK methods
- **No client hardcoding:** every agent reads from Notion at runtime
- **Approval gates are non-negotiable:** never auto-advance without a logged approval
- **Script subfolder structure:** `scripts/{seo,blog,care,visual,pipeline,onboarding,setup,util}/`
- **Default client in Makefile:** `CLIENT=summit_therapy` — override with `CLIENT=x`
